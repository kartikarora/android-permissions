Detail API

Endpoint

/detail/permission_name/json
OR 
/list/permission_name/xml

Response (e.g. CAMERA)

{
  "api_level": 1,
  "constant": "android.permission.CAMERA",
  "deprecated": 0,
  "desc": "Required to be able to access the camera device.\n\nThis will automatically enforce the manifest element for all camera features. If you do not require all camera features or can properly operate if a camera is not available, then you must modify your manifest as appropriate in order to install on devices that don't support all camera features.",
  "level": 1,
  "name": "CAMERA",
  "url": "https://developer.android.com/reference/android/Manifest.permission.html#CAMERA"
}
OR 
<CAMERA>
    <constant type="str">android.permission.CAMERA</constant>
    <name type="str">CAMERA</name>
    <level type="int">1</level>
    <url type="str">
        https://developer.android.com/reference/android/Manifest.permission.html#CAMERA
    </url>
    <deprecated type="int">0</deprecated>
    <api_level type="int">1</api_level>
    <desc type="str">
        Required to be able to access the camera device. This will automatically enforce the manifest element for all camera features. If you do not require all camera features or can properly operate if a camera is not available, then you must modify your manifest as appropriate in order to install on devices that don't support all camera features.
    </desc>
</CAMERA>

API Level: API level from which the permission is available.

Constant: String constant of the permission.

Deprecated: 1 if permission is deprecated, 0 otherwise

Desc: Description of the permission

Level: -1 - System | 0 - Normal | 1 - Dangerous

Name: Name of the permission

Url: Reference URL for the permission on Android Developer documentation
